Lab 2: Version Control

The second lab was all about version control software which I found very helpful to manage my CS118 coursework in the following weeks. The time of the lab was perfect for me to help me with my exercises in other modules. I used git on an example file ex1.html to test out this piece of version control software.

Exercises:

  1. The first thing I did was to initialise the git repository.
    git init
  2. After the creation of the git repo I added the example file to it.
    git add ex1.html
  3. After that I commited the changes:
    git commit -m "Added a new web page"
  4. Finally for the version control part of this lab I identified myself to the system:
    git config --global user.name "Keno Budde"
    git config --global user.email "k.budde(at)warwick.ac.uk"
  5. Moving on from this part I corrected the ex1.html and validated it using the online validator provided.
  6. After this part it was all about Cascadian Style Sheet, a possibility to style webpages. In this lab it was about using the id attribute to modify specific elements and the difference between relative and absolute positioning of elements.
  7. Finally I validated the css file created for this exercise.

Commands:

  • git init: Initialises a directory with git. An example might be
    git init cs133
  • git add: Adds a file to a repository. An example might be
    git add ./index.html
  • git commit: Has to be used with the -m flag to specify a description of the changes. An example would be:
    git commit -m "changed color of every h1 element to red"
  • git config: to edit the git config file, for example to specify an email adress:
    git config --global user.email "k.budde(at)warwick.ac.uk"
  • git diff: Lists the changes made relative to the index.
